# Windows任务计划程序配置指南

## 目的
配置每天自动执行的任务，保持Supabase数据库的活跃度，避免因长期不活跃而被暂停。

## 配置步骤

### 1. 打开任务计划程序
- 按下 `Win + R` 键打开运行对话框
- 输入 `taskschd.msc` 并按 Enter

### 2. 创建基本任务
- 在右侧 "操作" 面板中点击 "创建基本任务"
- **名称**: 输入 "Keep Supabase Active"
- **描述**: 输入 "每天自动请求数据库以保持Supabase活跃度"
- 点击 "下一步"

### 3. 触发器配置
- **任务开始**: 选择 "每天"
- 点击 "下一步"
- **开始时间**: 设置为适合的时间，例如 `09:00:00`
- **天数**: 保持默认值 1
- 点击 "下一步"

### 4. 操作配置
- **操作**: 选择 "启动程序"
- 点击 "下一步"
- **程序/脚本**: 浏览并选择批处理文件 `scripts\run-keep-active.bat`
  - 完整路径示例: `D:\edge下载\pack_project\projects\scripts\run-keep-active.bat`
- **起始于**: 输入项目根目录
  - 例如: `D:\edge下载\pack_project\projects`
- 点击 "下一步"

### 5. 完成配置
- 勾选 "当点击完成时，打开此任务的属性对话框"
- 点击 "完成"

### 6. 属性设置
- 在 "常规" 选项卡中，勾选 "使用最高权限运行"
- 在 "条件" 选项卡中，根据需要调整电源设置
- 在 "设置" 选项卡中，确保勾选 "如果任务失败，按以下间隔重试"
- 点击 "确定"

## 测试配置
- 右键点击创建的任务，选择 "运行"
- 检查 `logs\keep-supabase-active.log` 文件，确认任务执行成功

## 故障排除
1. **任务未执行**: 检查任务计划程序的历史记录
2. **执行失败**: 检查批处理文件路径是否正确
3. **数据库连接失败**: 检查 `.env.local` 文件中的配置

## 日志文件
执行日志保存在 `logs\keep-supabase-active.log`，包含每次执行的时间和状态。

## 手动运行
如果需要手动运行任务，可以直接双击 `scripts\run-keep-active.bat` 文件。
